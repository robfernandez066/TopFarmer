TASK-021: Connect FarmPlot activation to plant, harvest, and smart replant
Status: ready
Complexity: complex
Files: create res://scripts/farm/farm_plot_interaction_controller.gd and its Godot UID sidecar; create res://tests/farm_plot_interaction_controller_test.gd and its Godot UID sidecar; create res://tests/visual/farm_plot_interaction_preview.tscn, res://tests/visual/farm_plot_interaction_preview.gd, and its Godot UID sidecar
Acceptance:
- Create a reusable `FarmPlotInteractionController` Node that coordinates one or more existing FarmPlot instances without modifying FarmPlot, its scene, its visuals, its input-feedback behavior, or any asset
- Expose exactly `configure(now_utc_callable: Callable, selected_crop_id: StringName) -> bool`, `set_selected_crop_id(crop_id: StringName) -> bool`, `register_plot(plot: Node) -> bool`, and `unregister_plot(plot: Node) -> bool`; declare exactly `crop_planted(plot, crop_id)`, `crop_harvested(plot, crop_id)`, and `plot_action_ignored(plot, lifecycle_state)` signals
- `configure` accepts only a valid callable and either an empty crop ID or a crop ID present in `Balance`; invalid configuration returns `false` without replacing the last valid callable or selection. `set_selected_crop_id` uses the same empty-or-valid rule and changes no plot state by itself
- `register_plot` accepts only a live FarmPlot exposing the approved lifecycle and `activated` signal, connects that signal exactly once, and tracks the plot for growth updates. Duplicate registration returns `false` and cannot double-connect. `unregister_plot` cleanly disconnects and removes one registered plot; repeated or unknown unregister returns `false`
- Every controller process update calls the configured UTC callable at most once, requires a nonnegative integer result, and passes that same value to `update_growth` on every registered live plot. Invalid, missing, negative, or noninteger time output emits a developer diagnostic, mutates no plot, and does not fall back to an operating-system clock
- On one registered plot activation, obtain one valid supplied UTC value and apply exactly this policy. READY calls `harvest_crop` once and, on success, emits `crop_harvested`. GROWING changes no lifecycle value and emits `plot_action_ignored`. EMPTY with a remembered crop and an empty or matching selection calls `replant_last_crop`; EMPTY with a different valid selection calls `plant_crop` for that selection; EMPTY with no remembered crop calls `plant_crop` only when a valid selection exists. Successful planting or replanting emits `crop_planted`; a failed or unavailable action emits `plot_action_ignored`
- Harvest and replant remain two distinct player actions: tapping READY harvests and leaves EMPTY, with no automatic planting. The next single tap on EMPTY replants the remembered crop when selection is empty or still matches it, satisfying smart replant without reopening a crop menu
- Keep selection after harvest. If a future caller changes selection to another valid crop before the next EMPTY activation, that tap plants the newly selected crop instead of the remembered one; no selection UI is added in this task
- Each accepted FarmPlot gesture still produces only the existing press-scale and soft-click feedback from TASK-020. The controller adds no second sound, press animation, particle, popup, floating number, user-facing text, vibration, or duplicate activation
- The controller never reads growth duration directly, changes FarmPlot timestamps directly, calls an operating-system clock, invents remaining-duration state, or auto-replants from process updates. It uses only the approved public FarmPlot lifecycle methods and read-only queries
- Do not add inventory, seeds, costs, yields, Gold, XP, Starlight, rewards, save data, offline reconciliation, clock-integrity policy, production, UI, camera behavior, Android APIs, project settings, or main-scene changes in this task
- Create a 540x960 developer-only interaction preview with the approved meadow background, one existing FarmPlot centered and initially EMPTY, one controller configured for `sunwheat`, and a deterministic mutable test-clock callable. No user-facing game string is introduced
- In the preview, clicking the plot once plants Sunwheat and shows GROWING; clicking again while GROWING leaves it GROWING; pressing Space sets the mutable test clock to the plot's own `ready_at_utc` and the next process update shows READY; clicking READY harvests to EMPTY; the next single click replants Sunwheat as GROWING. Space is diagnostic-only and reads the stored ready timestamp rather than hardcoding a growth duration
- The preview prints developer-only lines for successful plant, ignored growing activation, successful harvest, and successful smart replant so the Human check can distinguish actions without adding UI. It never modifies the real system clock or writes save data
- The new headless test uses a fake callable and real Balance/FarmPlot scene to prove configuration validation, selection validation and replacement safety, single/duplicate registration, one UTC call per process update, invalid-time immutability, exact pre-deadline and ready transitions, the full tap plant/ignored-growing/harvest/one-tap-replant sequence, selection override with another valid crop, exact signals and counts, unregister behavior, freed-plot cleanup, no auto-replant, and unchanged TASK-020 feedback ownership
- All existing tests remain unchanged and pass. `project.godot`, FarmPlot files, every asset, every balance file, every localization file, and every file not listed above remain byte-for-byte unchanged except the append-only Coder report
